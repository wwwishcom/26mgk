## 02차시-1. 엑셀 데이터의 형태와 붙여넣기

### 엑셀에 입력 가능한 데이터 종류 (표시 형식이 [일반]일 때)

| 종류 | 특징 | 정렬 |
|---|---|---|
| 텍스트 | 문자 데이터 (계산 불가) | 왼쪽 정렬 |
| 숫자 | 숫자 데이터 (계산 가능) | 오른쪽 정렬 |
| 계산식 | 사칙연산/논리연산 등 수식의 결과값 | 결과가 텍스트면 왼쪽, 숫자면 오른쪽 |
| 함수 | 데이터 입력 시 특정 결과 출력 | 결과값 형태에 따름 |

- **서식**: 글꼴 서식(색, 크기, 굵게, 기울임, 밑줄), 셀 서식(배경색, 테두리 등)
- **엑셀의 모든 셀에는 이미 서식이 적용되어 있음** (기본값: 맑은고딕 / 11pt / 검정 / 테두리X / 배경색X)

### 붙여넣기 핵심 개념

- `Ctrl+C` → `Ctrl+V` = **"모두 붙여넣기"** → 값/수식뿐 아니라 **서식까지 전부** 복사됨
  - ⚠️ 원치 않는 테두리, 배경색 등의 서식이 함께 붙여넣어질 수 있음
- 상황에 따라 **따로따로** 붙여넣어야 함:
  - **수식 붙여넣기**: 서식 제외, 수식 그대로 붙여넣기
  - **값 붙여넣기**: 서식 제외 + 수식을 값으로 변경해 붙여넣기
  - **서식 붙여넣기**: 값/수식 제외, 서식만 붙여넣기
- 단축키: `Ctrl+Alt+V` (선택하여 붙여넣기)

## 02차시-2. 엑셀의 참조

- 엑셀은 하나의 행과 하나의 열이 교차하는 지점, 즉 **'셀(Cell)'**에 데이터를 입력하는 프로그램
- **행은 상하로, 열은 좌우로** 이동(변경)됨

### 참조의 3종류 (F4 키로 전환)

| 참조 종류 | 특징 | F4 누른 횟수 | 예시 |
|---|---|---|---|
| 상대 참조 | 복사/붙여넣기 시 참조된 셀이 **함께 이동** | 4번째 (원상복귀) | `C5:C12` |
| 절대 참조 | 복사/붙여넣기 해도 참조된 셀이 **이동하지 않음** | 1번 | `$C$5:$C$12` |
| 혼합 참조 (행 고정) | 행은 절대참조(위/아래 이동 X), 열은 상대참조(옆으로 이동 O) | 2번 | `C$5:C$12` |
| 혼합 참조 (열 고정) | 행은 상대참조(위/아래 이동 O), 열은 절대참조(옆으로 이동 X) | 3번 | `$C5:$C12` |

> ⭐ **시험 포인트**: F4는 **상대 → 절대 → 혼합(행고정) → 혼합(열고정) → 다시 상대**로 순환. `$`는 뒤에 오는 행 번호나 열 문자를 고정.

## 함수(函數)의 기본 개념

- 입력값 'x'에 대응하는 하나의 출력값 'y'를 가지는 관계
- 복잡한 수식이나 계산을 간단한 명령어로 사용할 수 있게 하는 도구
- 함수에 **[데이터를 입력]**하면 **[결과값을 출력]**
- `[ ]`(대괄호)가 있는 인수는 **생략 가능**
- 모든 인수는 **콤마(,)로 구분**

| 예시 | 인수 개수 |
|---|---|
| `=TODAY()` | 0개 |
| `=SUM(C5:C12)` | 1개 |
| `=COUNTIF($N:$N,R5)` | 2개 |
| `=IF(F5=$R$7,$T$7,$T$8)` | 3개 |
| `=VLOOKUP($D6,'KPI2019(DETAIL)'!$D:$P,E$3,0)` | 4개 (range_lookup은 [ ]로 생략 가능) |

- **SUM 함수**: 선택한 데이터 범위의 **합계**
- **AVERAGE 함수**: 선택한 데이터 범위의 **평균**
- `[Tab]`키: 함수명 자동완성

## 06차시. IF 함수와 중첩으로 데이터 분류하기

- 부등호/등호(`>, <, =`)를 활용해 조건을 가정하고, 데이터를 조건에 **만족하는 값**과 **만족하지 않는 값**으로 분류
- 구조: `=IF(조건, 조건을 만족할 경우 표시할 값, 조건을 만족하지 않을 경우 표시할 값)`

### ⭐ IF 중첩의 핵심 공식 (최다 출제 포인트)

> **IF 함수를 N번 중첩 사용하면 데이터가 N+1가지로 구분됨**

- IF 함수 2번 중첩 → 데이터 3가지로 구분 (예: Platinum / Gold / Silver)
- IF 함수 3번 중첩 → 데이터 4가지로 구분

## 07차시. COUNT 함수 계열로 데이터 개수 세기

| 함수 | 세는 대상 |
|---|---|
| **COUNT** | 범위에서 **숫자 데이터**가 들어 있는 셀의 개수 |
| **COUNTA** | 범위에서 **비어있지 않은 모든 셀**(데이터 종류 무관)의 개수 |
| **COUNTBLANK** | 범위에서 **비어있는 셀**의 개수 |
| **COUNTIF** | 범위에서 **1개 조건**을 만족하는 셀의 개수 |
| **COUNTIFS** | 범위에서 **2개 이상 조건을 동시에** 만족하는 셀의 개수 |

> ⚠️ **시험 포인트**: 함수 안에서 부등식을 조건으로 사용할 때는(COUNTIF 등) 일반 수식과 달리 **따옴표(" ")를 붙여야 함**
> 보통 조건 1개는 COUNTIF, 2개 이상은 COUNTIFS로 알려져 있지만 상황에 따라 유연하게 사용 가능

## 08차시. VLOOKUP 함수로 데이터 불러오기

- **공통 기준열**을 기준으로 n번째 있는 데이터를 불러오는 함수
- **V**ertical(수직의) → '열'을 중심으로 / **Look up**(찾다) → 데이터를 찾아옴
- HLOOKUP보다 VLOOKUP이 많이 쓰이는 이유: 인간이 데이터를 **"열"별로 정리**하는 습관이 있기 때문
- **원리**: 공통 기준열을 1열이라 가정하고, n번째 있는 열의 데이터를 불러옴

### 인수 구조
`=VLOOKUP(찾을 기준 데이터, 불러올 데이터의 범위(공통 기준열부터~), 불러올 데이터의 열 번호(기준열을 1열로 했을 때), [range_lookup])`

- **range_lookup**: `0`(FALSE) = 정확히 일치할 때만 / `1`(TRUE) = 근사치여도 허용

## 09차시. VLOOKUP 함수의 활용

### #1. 열 번호 참조하기
- 기존: `col_index_num`을 직접 숫자로 입력 → 다른 데이터를 불러올 때마다 열 번호를 일일이 바꿔야 하는 비효율 발생
- 개선: `col_index_num`에 숫자가 아닌 **열 번호가 입력된 셀을 참조** → 복사/붙여넣기만 하면 자동으로 열 번호 변경

### #2. MATCH 함수 대입하기
- 열 번호를 참조하더라도 처음에 몇 번째 열인지 직접 세야 하는 번거로움 → **MATCH 함수**를 `col_index_num`에 대입하면 자동 계산/변경 가능
- MATCH 함수는 찾고 싶은 값이 한 행/열에서 몇 번째에 있는지 숫자로 알려줌

### #3. 기준 열에 중복값이 있는 경우
- VLOOKUP은 기준열에 중복값이 있으면 **가장 위에 있는 데이터의 정보만** 불러옴
- 해결책: 중복값이 없는 **새로운 기준열**을 생성해 VLOOKUP 사용

## 10차시. INDEX & MATCH 함수 활용

### MATCH 함수
- 찾고 싶은 값이 **한 행 또는 한 열(단일 범위)**에서 몇 번째에 있는지 **숫자**로 알려주는 함수
- 결과는 반드시 숫자(몇 번째인지)
- 구조: `=MATCH(내가 찾고 싶은 값, 내가 찾고 싶은 값이 포함된 단일 열 또는 행 범위, [정확히 일치 또는 근사치 여부])`
- 인수는 `lookup_value`, `lookup_array`, `[match_type]` 총 3개
- `match_type`: -1, 0, 1 중 입력

> ⚠️ **시험 포인트**: MATCH의 두 번째 인수(lookup_array)에 **2차원 범위(예: A1:C3처럼 여러 행×여러 열)를 입력하면 오류** 발생 — 반드시 단일 행 또는 단일 열이어야 함

### INDEX 함수
- 특정 범위에서 **행 번호와 열 번호**로 원하는 데이터를 불러옴
- 구조: `=INDEX(데이터의 전체 범위, 찾고 싶은 데이터의 범위 내 행 번호, 찾고 싶은 데이터의 범위 내 열 번호)`
- 인수 순서는 **범위 → 행 번호 → 열 번호** (열 번호가 먼저 아님!)
- 실무에서는 행 번호/열 번호 자리에 **MATCH 함수를 중첩(INDEX-MATCH 조합)**하여 자동으로 위치를 찾도록 많이 사용

## 11차시. SUMIF(S) 함수로 데이터 합계 구하기

### SUMIF 함수
- 특정 조건을 만족하는 데이터의 **합계** 계산
- 구조: `=SUMIF(조건 범위, 조건, 더할 값들의 범위)`

### SUMIFS 함수
- **2개 이상의 조건을 동시에** 만족하는 데이터의 합계 계산
- 구조: `=SUMIFS(더할 값들의 범위, 조건 범위1, 조건1, 조건 범위2, 조건2, …)`

> ⭐ **시험 포인트**: **SUMIFS는 SUMIF와 다르게 "더할 값들의 범위"를 가장 먼저 지정**해야 함 (SUMIF는 조건 범위, 조건이 먼저 오고 더할 범위가 마지막)
> 보통 조건 1개는 SUMIF, 2개 이상은 SUMIFS로 알려져 있지만 상황에 따라 유연하게 사용 가능

## 12차시. IFERROR 함수로 오류값 처리하기

### 엑셀의 주요 오류 종류

| 오류 | 원인 |
|---|---|
| `#DIV/0!` | 나누는 수(분모)가 0일 때 |
| `#N/A` | 찾을 값이 없거나, 배열 함수에서 범위의 인수가 서로 일치하지 않을 때 (VLOOKUP 등에서 자주 발생) |
| `#NAME?` | 함수명이 잘못되었거나 잘못된 인수가 입력됐을 때 |
| `#NULL!` | 범위 연산자 사용이 잘못되었거나 교차하지 않는 영역을 참조할 때 |
| `#NUM!` | 함수의 인수나 수식이 잘못 입력되었을 때 |
| `#REF!` | 참조된 셀 주소가 잘못되었거나 삭제되었을 때 |
| `#VALUE!` | 논리 값/숫자가 필요한 수식에 텍스트를 입력했거나, 배열 수식을 Ctrl+Shift+Enter로 입력하지 않았을 때 |
| `#####` | 셀 값보다 열 너비가 좁을 때 |

### IFERROR 함수
- 입력된 수식/함수의 결과가 **오류가 아니면** 수식의 결과값을 그대로 출력
- 결과가 **오류값이면** 내가 원하는 값을 출력 가능
- 구조: `=IFERROR(수식, 오류가 발생했을 때 표시할 값)`
- 예: 오류 시 빈 칸으로 표시 → `=IFERROR(A1/B1, "")`

## 13차시. FIND 함수로 문자 데이터 위치 파악하기

### FIND 함수
- **긴 텍스트에서 특정 단어나 문장이 시작하는 위치**를 숫자로 출력
- **띄어쓰기까지 포함**하여 문자 수를 세며, **대소문자를 구분**함
  - 대소문자를 구분하지 않아야 할 때는 **SEARCH 함수** 사용
- 구조: `=FIND(찾을 텍스트, 긴 텍스트, [문자열을 찾기 시작할 위치])`
  - 마지막 인수를 생략하면 텍스트의 첫 번째 문자부터 검색 시작

### 활용: 특정 단어 포함 여부 파악
- 결과로 **숫자가 나오면** 특정 단어가 포함된 것, **오류가 나면** 포함되어 있지 않은 것으로 해석

## 14차시. LEFT RIGHT MID 함수로 데이터 추출하기

### LEFT / RIGHT 함수
- 텍스트의 **가장 왼쪽 / 오른쪽**부터 원하는 문자열까지 추출
- 구조: `=LEFT(전체 텍스트, 불러올 문자열 수)` / `=RIGHT(전체 텍스트, 불러올 문자열 수)`

### MID 함수
- 텍스트의 **중간 시작 위치**부터 원하는 문자열까지 추출
- 구조: `=MID(전체 텍스트, 불러올 문자열의 시작 위치, 불러올 문자열 수)`

> ⚠️ **시험 포인트**: MID 함수의 인수 순서는 **text(전체 텍스트) → start_num(시작 위치) → num_chars(문자 수)** 순서 (start_num이 먼저가 아님)

## 19차시. 필터와 고급 필터

### 필터
- **필터링**(필요한 데이터만 선택 or 필요하지 않은 데이터 제외) 기능 + **데이터 정렬**(오름차순/내림차순) 기능

> ⭐ **시험 포인트**: 조건을 한 행에 쓰면 AND(모두 만족), 여러 행에 나눠 쓰면 OR(하나 이상 만족)

## 21차시. 차트와 그래프 그리기

### 차트 종류별 특징
| 차트 | 표현 방식 |
|---|---|
| 막대 그래프 | 막대의 높낮이로 수치의 크고 작음/많고 적음 표현 |
| 선형 그래프 | 선의 높낮이로 수치의 높고 낮음 표현 |

| 콤보형 차트 | 막대형/꺾은선형 등 2개 이상의 그래프를 하나의 차트에 표현 |
| 거품형 차트 | 숫자의 크기·비율을 거품(원)으로 표현 (거품 크기, 위치, 색으로 정보 표현) |
| 폭포형 차트 | 데이터의 증가·감소분만큼을 막대 그래프로 표현 (엑셀 2016 이상에서 사용 가능) |

### 콤보형 차트 만들기
1. 데이터 범위 선택 → [막대형 차트] 삽입
2. 그래프의 막대 선택 → 마우스 오른쪽 클릭 → [계열 차트 종류 변경]
3. 차트를 변경할 계열의 차트 종류 변경 → 필요 시 [보조축] 선택

### 거품형 차트 데이터 선택
- 다른 차트와 달리 **먼저 빈 차트를 생성한 후** [데이터 선택]에서 데이터를 입력하는 것이 효율적
- 차트 선택 → 마우스 오른쪽 클릭 → [데이터 선택] → 계열 이름, 계열 X값, 계열 Y값, 계열 거품 크기 순서로 입력

### 폭포형 차트 계산 방식
- 1차 년도는 기존 데이터 그대로 사용
- 2차 년도부터는 `[=해당 년도 - 직전 년도]`로 증감분을 계산해서 표현

## 22차시. 조건부 서식의 이해와 활용

- 특정한 조건 규칙을 만족하는 데이터에만 **자동으로 서식이 적용**되도록 하는 기능

### 조건부 서식 규칙 6가지

| 규칙 | 설명 | 예시 |
|---|---|---|
| **셀 값을 기준으로 모든 셀의 서식 지정** | 값의 상대적 크기에 따라 시각적으로 표현 (2가지 색조, 데이터 막대, 아이콘 집합) | 값이 클수록 진한 색/긴 막대 |
| **다음을 포함하는 셀만 서식 지정** | 특정 조건(이상/이하/사이/텍스트 포함 등)을 만족하는 셀에만 서식 | 확진자 수가 1,000명 이상인 지역 |
| **상위 또는 하위값만 서식 지정** | 상위 10개(%) 또는 하위 10개(%)에 해당하는 데이터에만 서식 | 확진자 수 Top 10 |
| (평균 기준) | 선택한 데이터의 평균보다 크거나 작은 데이터에만 서식 | 평균보다 작은 확진자 수 |
| **고유 또는 중복값만 서식 지정** | 고유(1개만 있는) 또는 중복(2개 이상)된 데이터에만 서식 | 동일한 확진자 수를 기록한 월/지역 |
| **수식으로 서식 지정** | 임의의 수식 결과가 참(TRUE)이 되게 하는 데이터에만 서식 | 1월이 4개월 중 최대치(Max)인 경우 |

> ⭐ **시험 포인트**: "○○ 이상/이하인 경우에만 서식 적용"처럼 **구체적인 임계값 조건**이 나오면 → **"다음을 포함하는 셀만 서식 지정"** 규칙 사용

### 자료 전달 시 유의사항 (엑셀 파일 공유 매너)
- 화면 비율 설정 75~100%
- 마우스 커서는 항상 A1셀(문서의 가장 위)에 위치
- 필요한 경우 구분자를 중심으로 틀 고정
- Sheet가 많은 경우(10개 이상) 빈 Sheet로 구분해주기
- Sheet 이름 반드시 변경하기 (Sheet1, Sheet2 X)

## ⭐ 시험 자주 나오는 함정 총정리

| 항목 | 함정 | 정답 |
|---|---|---|
| 빠른 실행 도구 모음 단축키 | Ctrl+숫자키 | **Alt+숫자키** |
| 숫자 표시형식 단축키 | 통화로 변경 | 통화 아니고 **숫자(콤마)** 표시형식 |
| 셀 내 줄 바꾸기 | Ctrl+Enter | **Alt+Enter** |
| FIND 함수 | 대소문자 구분 안 함 | **대소문자 구분함** (구분 안 하려면 SEARCH) |
| MID 함수 인수 순서 | start_num, text, num_char | **text, start_num, num_chars** |
| INDEX 함수 인수 순서 | 범위, 열 번호, 행 번호 | **범위, 행 번호, 열 번호** |
| MATCH 함수 lookup_array | 2차원 범위(A1:C3) 가능 | **단일 행 또는 단일 열만 가능** |
| SUMIFS 인수 순서 | 조건 범위가 먼저 | **더할 값의 범위가 가장 먼저** |
| 피벗 테이블 | 원본 변경 시 자동 반영 | **자동 반영 안 됨, [새로 고침] 필요** |
| IF 중첩 | N번 중첩 = N가지 분류 | **N번 중첩 = N+1가지 분류** |
